# BeaconAI — Known Issues & Engine Behavior Registry

**Purpose:** single source of truth for open bugs, edge cases, dormant behavior, and accepted operator-discipline gaps in the V2 engine. Updated whenever a ticket lands a non-obvious caveat or a new finding surfaces (e.g., G-1 supplements bug list).

## How to use this file

- **Adding entries:** every new issue gets an ID (`KI-<n>`), a one-line title, status, source, and a short "why it matters / how to apply" body. Append to the right category below.
- **Status values:** `open` (not fixed, no ticket) / `tracked` (Sprint N ticket scoped) / `accepted` (operator discipline; no fix planned) / `resolved` (fixed; keep entry for one sprint then archive).
- **Sources:** cite the summary file or commit where the issue was first documented.
- **When tickets land:** if a ticket resolves an entry here, flip status to `resolved` and add the resolving commit hash. Archive after one sprint.
- **What goes IN:** bugs, edge cases, engine behavior quirks, dormant code paths, contract risks, operator-discipline gaps that a future agent or merchant could trip on.
- **What stays OUT:** completed work (see summary files), founder strategy decisions (see `memory.md` D-1 through D-8), Phase 9+ roadmap items (see implementation plan).
